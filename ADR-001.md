# ADR-001: Escolha da Stack Java/Spring Boot para o NanoLink

*   **Status:** Proposto
*   **Data:** 2026-01-08
*   **Agente:** @Architect

## Contexto
Para o desenvolvimento do MVP do encurtador de URLs "NanoLink", é necessária uma stack de backend que seja robusta, rápida de desenvolver, escalável e de fácil manutenção. O projeto requer a criação de uma API REST, lógica de negócio para geração de códigos e persistência de dados.

## Decisão
Adotaremos a seguinte stack tecnológica:
*   **Linguagem:** Java (versão 17 ou superior)
*   **Framework:** Spring Boot 3
*   **Ferramenta de Build:** Apache Maven
*   **Componentes Spring:**
    *   **Spring Web:** Para a criação dos endpoints REST (`/api/v1/shorten` e `/{shortCode}`).
    *   **Spring Data JPA:** Para a camada de persistência com o banco de dados.
    *   **H2 Database:** Como banco de dados em memória para o MVP, visando simplicidade e rapidez no setup inicial.


## Consequências
### Vantagens
*   **Ecossistema Maduro:** O Spring Framework oferece um ecossistema completo e testado pela indústria.
*   **Desenvolvimento Rápido:** O Spring Boot acelera a configuração e o desenvolvimento de novas aplicações.
*   **Segurança de Tipos:** O Java oferece um sistema de tipos forte, que ajuda a evitar erros em tempo de execução.
*   **Comunidade e Documentação:** Vasta documentação e uma comunidade ativa para suporte.
*   **Performance:** A JVM é altamente otimizada e oferece excelente performance para aplicações web.

### Desvantagens
*   **Consumo de Memória:** Aplicações Spring Boot podem ter um consumo de memória inicial maior em comparação com alternativas em Go ou Node.js.
*   **Verbosity:** O Java pode ser mais verboso que outras linguagens.
